Exploring JavaScript vs Ruby

7/10/15

I'm assuming that all of you guys are familar with the basics of both Ruby and Javascript. These languages can be used in a very similar fashion. One of the most important aspects of coding (in any language) is the use of key and value pair to store, access, and compute data. In Ruby this is done threw hashes, while in Javascript this is done threw objects. The following post will go into the syntax of both languages and compare how they similar and different

What are JS objects?

Objects are simply variables that stored in a key/ value manner. To get the value of the object you "call" the key. If you confused, checkout this brief example of basic JS syntax below,

var x = { a: 1, b: 2, c: 3 };

The code above simply created an object "x" that stores 3 key/ value pairs. To you can access the value of each of these pairs a number of ways.

console.log(x.a); # print "1" to the screen console.log(x['b']); # print "2" to the screen

although both of these methods of accessing the value are valid, the more common approach is the object literal one (x.a).

If you want to delete a pair in this object, you should do the following

delete x.c;

The code above will delete the c key and value from this object and can thus no longer be accessed

What are Ruby hashes?

On a basic level ruby Hashes and JS objects serve the same purpose, to store key/ value pairs for accessing data, but their syntax is different. Below I will be doing the same example above, just in Ruby as opposed to JS.

x = {} x['a'] = 1 x['b'] = 2 x['c'] = 3

To access these values you must use the second way we used to access the value in JS

puts x["a"] # print "1" to the screen puts x["b"] # print "2" to the screen puts x["c"] # print "3" to the screen

To delete objects you have to call a method on the hash. See the following for further clarification

x.delete("c")

Differences between JS and Ruby

Outside of the basic syntax it is hard to see how these two differ. To me, the Javascript Object stored system seems much more intiutive than the ruby's hashes.
