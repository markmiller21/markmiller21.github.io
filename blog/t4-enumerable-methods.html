Map Enumerable

6/14/15

The map enumerable is one of the most useful and powerful built-in methods that we have in ruby. If you are yet to hear of the map method, then you are probably familar with the each method. What each does, is allows you to iterate through each data point in an array of hash. Map works similarly to each, but differs in the way it returns values. Each doesn't return any values, while map does. In reality, each merely exists for the side effects from the execuation of the block. Map changes the content for the array or hash. Below is an example of how to use the map method,

place_array = ["dc", "nyc", "sf", "no"] #=> ["dc", "nyc", "sf", "no"] place_cap_array = place_array.map {|place| place.upcase} #=> ["DC", "NYC", "SF", "NO"] place_array #=> ["dc", "nyc", "sf", "no"] place_cap_array #=> ["DC", "NYC", "SF", "NO"]

Now that you're familar with the map method, it's time to get into the pitfalls of map. Map only cares about the return value of the array of hash. This may sound like a good thing in theory, but take a look at the block of code below to see how this method could have adverse effects.

array = [1,2,3,4,5] result = array.map {|n| puts n ** 2}

You would think the resulting return results line would yeild

results #=> [1,4,9,16,25]

But the actual return would is

results #=> [nil, nil, nil, nil, nil]

This is because puts always returns nil, despite what it prints to the screen. This is a minor pitfall of the map method that shows map is not always the best method to use.
